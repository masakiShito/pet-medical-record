# 用語定義

Pet Medical Record App（ペットカルテアプリ）

---

## 1. 目的

本書は、**ドメイン用語の表記揺れを防ぎ、開発者・AI間での認識を統一する**ことを目的とする。

---

## 2. ドメイン用語

### 2.1 基本エンティティ

| 用語（日本語） | 用語（英語） | 定義 |
|---|---|---|
| ユーザー | User | 飼い主（MVPでは1ユーザー想定） |
| ペット | Pet | 管理対象の動物 |
| 記録 | Record | 日付単位の健康記録（親） |
| 体重記録 | Weight Record | 体重測定の記録 |
| 投薬記録 | Medication Record | 薬の投与記録 |
| 通院記録 | Vet Visit Record | 動物病院受診記録 |

### 2.2 記録関連

| 用語 | 定義 |
|---|---|
| 記録日 | 記録が該当する日付（recorded_on） |
| 体調レベル | 1〜5の5段階評価 |
| 食欲レベル | 1〜5の5段階評価 |
| 便の状態 | 1〜5の5段階評価 |

### 2.3 UI用語

| 用語 | 定義 | 備考 |
|---|---|---|
| カード | 情報を囲むUIブロック | ペット一覧、記録一覧で使用 |
| トースト | 一時的なメッセージ表示 | 成功・エラー通知 |
| モーダル | オーバーレイ表示のダイアログ | 削除確認など |

---

## 3. 表記ルール

### 3.1 統一表記

| NG | OK |
|---|---|
| pet | ペット |
| カルテ | 記録 |
| 病院 | 動物病院 |
| 先生 | 獣医師 |
| お薬 | 薬 |

### 3.2 日付・時刻

- 記録日: `YYYY-MM-DD`
- 作成日時: `YYYY-MM-DD HH:mm:ss`

---

## 4. 論理削除

| 用語 | 定義 |
|---|---|
| 論理削除 | is_deleted フラグで管理 |
| 物理削除 | 禁止（MVP範囲外） |

---

## 5. 状態・ステータス

| 状態 | 値 | 説明 |
|---|---|---|
| 正常 | ok | API・DB接続正常 |
| エラー | error | 異常状態 |
| 確認中 | checking | ヘルスチェック中 |

---

## 6. 禁止用語

以下の用語は使用しない：

- 患者（ペットを指す場合）
- カルテ（記録を使用）
- マスター（ユーザー・飼い主を使用）
-  API・DBは複数形を使用する（pets, records）
- UI文言は日本語自然表現を優先
