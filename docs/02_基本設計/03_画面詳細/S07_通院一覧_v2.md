# S07 通院一覧 画面詳細（v2）

## 1. 画面概要
- **目的**: ペットの通院（診療）を時系列で一覧化し、詳細閲覧・追加・編集に迷わず到達できるようにする。
- **URL**: `/pets/:petId/vet-visits`

---

## 2. 一覧の基本仕様
- ソート: 新しい順（受診日降順）
- 表示単位: 1通院 = 1カード（要点表示）
- ページング: MVPは無くてもOK（将来追加）

---

## 3. 画面項目一覧

| No | 物理名 | 項目名 | 種類 | 必須 | 仕様 | 備考 |
|---:|---|---|---|---|---|---|
| 1 | filters.keyword | キーワード | 入力 | 任意 | 病院名/診断/メモを対象 | 将来/任意 |
| 2 | list[] | 通院カード一覧 | 表示 | - | 受診日/病院/診断/費用 | |
| 3 | list[].visited_on | 受診日 | 表示 | - | YYYY/MM/DD | ※visited_on採用時 |
| 4 | list[].hospital_name | 病院名 | 表示 | - | 最大30文字目安 | |
| 5 | list[].diagnosis | 診断 | 表示 | - | 省略表示（2行） | |
| 6 | list[].cost_yen | 費用 | 表示 | 任意 | ¥1,234 | |
| 7 | actions.new | 通院追加 | ボタン | - | S08へ | 画面右下FAB推奨 |
| 8 | actions.detail | 詳細へ | リンク | - | S09へ | カードクリック |

---

## 4. 操作フロー
### 4.1 初期表示
1. petIdの通院一覧取得
2. カード表示

### 4.2 追加
1. 「通院追加」クリック
2. S08へ遷移

---

## 5. エラーハンドリング
- 一覧取得失敗: リトライ表示
- データ0件: 空状態（「通院を追加」CTAを強く）

---

## 6. UXメモ
- 空状態でも「最初の通院を追加」ボタンを中央に配置
- カードは「日付→病院→診断→費用」の順で視線誘導
